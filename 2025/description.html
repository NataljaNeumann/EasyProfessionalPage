Basic Structure of HTML Document

<!DOCTYPE html>
<html lang="..your language abbreviation..">
<head>
    <meta charset="UTF-8">
    <title>..Your title for the page../title>
    <link rel="stylesheet" href="epp.css" />
</head>
<body dir="ltr">
    <div class="main">

    </div>
</body>
</html>

To create an empty HTML document with the needed structure and styles, follow these steps:

  • Start with the doctype declaration to define the document as HTML5
  • tag and specify the language (e.g.,
    • en for English
    • fr for French
    • es for Spanish
    • pt for Portuguese
    • de for German
    • it for Italian
    • zh for Chinese
    • ja for Japanese
    • ru for Russian
    • ar for Arabic
    • hi for Hindi
    • and so on, you can find a complete list in Internet...
  • Include a header specification, including following elements:
    • Character encoding (usually UTF-8)
    • Title for the page
    • Link to an external CSS file named epp.css
  • Include a body section with:
    • Set the text direction in the dir attribute, e.g. "ltr" for left-to-right or "rtl" for right-to-left
    • Optionally, set the language of the body using the lang attribute
    • Include a <div> element with style="main" to contain the main content of the page
  • Close the HTML document by writing </div><body><html>

What the Page Will Look Like

With the provided CSS variables and styles, here's how the page will appear:
  • Body uses the primary font defined by "--font-primary" variable
  • Background color that is to the left and right of the main area is set to "--color-box-gray-bg"
  • Text color is "--color-text-main"
  • No default margin or padding (clean edge-to-edge layout)
Main Content Area has
  • Background color is set to "--color-bg-main"
  • Width is limited to "--width-desktop" (e.g., 1200 pixel or similar)
  • Horizontally centered
  • Padding inside the main area is defined by "--spacing-element" variable
  • Takes up at least the full height of the viewport, ensuring it fills the screen even with no content

You can customize the fonts used in your web page by modifying CSS variables in the epp.css file. If you include @font-face rules in your CSS, you can use custom fonts as well. The page uses the following font variables:

--font-primary: Main font for body text and most elements.
--font-secondary: Secondary font for headings and special elements.

By defining these variables, you can easily change the overall typography of your web page, however you can also change fonts of specific elements, by specification of the fonts in the styles of these elements. You can also define a font by using another font variable. The stylesheet already defines some fonts that you can use, when you include and install the corresponding fonts. Please have a look at the licence of the fonts.

Finally you can use the font variables on the elements themselves, by adding style attribute

/* Using a variable for defining another variable in CSS */
  --font-primary: var(--font-playfair);

/* specification of specific font for main header in CSS */
h1 {
  font-family: var(--font-playfair);
  font-weight: bold;
}

<!-- specification of specific font for a div in HTML,
    in this case the primary font  -->

<div style="font-family: var(--font-primary);">
    This text uses the primary font.
</div>

Size variables

In CSS, em and rem are relative units used for sizing elements like fonts, margins, and padding. Both are scalable, making them ideal for responsive design.

The em unit is relative to the font size of the element’s parent. For example, if a parent element has a font size of 16px, then 1em equals 16px, and 2em equals 32px. However, since em is relative to its parent, nested elements can compound sizes, which may lead to unexpected results if not managed carefully.

The rem unit stands for “root em” and is always relative to the root element’s font size—typically the <html> element. If the root font size is 16px (the browser default), then 1rem equals 16px regardless of nesting. This makes rem more predictable and easier to maintain in large projects.

Typical Sizes by Device (assuming 1rem = 16px):

  • Mobile phones: 14–16px (0.875rem–1rem)
  • Tablets: 16–18px (1rem–1.125rem)
  • Desktops: 16–20px (1rem–1.25rem)

Using rem for base font sizes and em for spacing or component-specific scaling is a common best practice. This approach ensures consistency while allowing flexibility across different screen sizes and user settings, however the CSS uses mostly rem units for spacings, so the spacings don't change with font size

Layout & Spacing Variables

  • --spacing-element - General spacing between elements (e.g. margins, vertical rhythm). 1.875rem is equivalent to 30px if root font size is 16px.
  • --padding-box - Internal padding for containers or boxes. 1.5rem is about 24px.
  • --grid-box-padding - Padding inside grid items or cards. 1rem is about 16px.
  • --grid-box-gap - Gap between grid items. Helps define spacing between grid cards.
  • --width-desktop - Max width for desktop layouts. 60rem ~ 960px.
  • --width-tablet - Max width for tablet layouts. 40rem ~ 640px.
  • --width-box-grid - Width of a grid cards container or content box. 29rem ~ 464px.
  • --width-half-column - Used for half-width elements, aligned left or right.
  • --line-thin - Thin border or divider line. 0.125rem ~ 2px.
  • --line-medium - Medium-weight line. 0.187rem ~ 3px.
  • --grid-box-radius - Border-radius for grid boxes or cards. 0.5rem ~ 8px.
  • --rounded-radius - Generic rounded corners for UI elements. Likely reused across buttons, inputs, etc. 0.5rem ~ 8px
  • --grid-box-width - Fixed width for individual grid cards. 15rem ~ 240px

You can change the spacings, as you want, as well as any other content, since this CSS is provided under MIT license.

Color variables

Modern CSS supports automatic adaptation to the user's preferred color scheme using the prefers-color-scheme media query. This allows developers to define separate styles for light and dark modes.

@media (prefers-color-scheme: light) {
  body {
    background-color: white;
    color: black;
  }
}

Light Mode

In light mode, the background is typically light (e.g., white or soft gray), and text is dark for readability. It's the default for most websites and resembles usual experience with black text on white paper that users have in everyday life.

@media (prefers-color-scheme: dark) {
  body {
    background-color: black;
    color: white;
  }
}

Dark Mode

Dark mode uses dark backgrounds (e.g., black or deep gray) with light text. It reduces eye strain in low-light environments and saves battery on OLED screens. Some users also prefer dark color schemes. 

@media print {
  body {
    background: none;
    color: black;
    font-size: 0.75rem;
  }

  nav, footer {
    display: none;
  }
}

Print Media Styling

CSS also allows customization for printed output using the @media print query. This ensures that pages look clean and readable when printed.
Backgrounds: Many browsers ignore background colors/images when printing unless explicitly enabled. Use background: none; or avoid heavy backgrounds.
Color to Grayscale: Colors may be converted to grayscale when printing. Avoid relying on color alone to convey meaning (e.g., red for errors).
Ink Efficiency: Use high-contrast, ink-friendly colors. Avoid saturated backgrounds or gradients.

Standard HTML Elements

Below you can find a standard list of html tags that you can use in your pages without specification of classes and styles. These will be automatically styled to the theme of the page. More examples of HTML tags can be found at w3schools

Headings and Text Structure

TagDescription
<h1> .. <h4>Headings used to structure content hierarchically. <h1> is the most important (e.g., page title), followed by <h2> to <h4> for subheadings.
<p>Paragraph element for blocks of text.
<b>Bold text for emphasis (visual only, no semantic meaning).
<i>Italic text for emphasis or stylistic use (also visual only).
<pre>Preformatted text block that preserves whitespace and line breaks — often used for code or ASCII art.

Lists

TagDescription
<ul>Unordered list — displays items with bullets.
<ol>Ordered list — displays items with numbers.
<li>List item — used inside <ul> or <ol>

Tables

TagDescription
<table>Defines a table structure.
<colgroup>Table column specification section
<col>Table columns specification, used inside <colgroup>
<thead>Table header section — usually contains column titles.
<tbody>Table body — contains the main rows of data.
<tr>Table row — used inside <thead> or <tbody>
<td>Table cell — holds data within a row.

Links and Media

TagDescription
<a>Anchor tag — creates hyperlinks to other pages, sections, or files.
<img>Image tag — embeds images using the src attribute.
<hr>Horizontal rule — creates a visual divider line across the page.

Those are the common elements that are used on pages and are recognized also by old browsers. Even if old browsers won't be able to present the page as it has been designed, still they will recognize the elements and will present them in a simplified way.

Image classes

The CSS defines classes for image elements that you can use for modification of the appearance of image elements. After the discussion of image classes will will also discuss image boxes that allow you to lay other content over the image. Image classes aren't compatible with image boxes. For each picture you need to use the one or the other. 

<img src="image.jpg" alt="Full width image" class="img-full">

.img-full

Purpose: Displays the image at full width, extending slightly beyond the container’s padding.

Behavior:
Expands the image to be slightly wider than its container to the border of the outer area.
Removes horizontal margins to align with the container edges.
Automatically adjusts height to maintain aspect ratio.

<img src="image.jpg" alt="Left-aligned image" class="img-left">

.img-left

Purpose: Displays a half-width image floated to the left (or start of the inline direction).

Behavior:
Floats the image to the left side of the container.
Sets the width to a predefined half-column value.
Adds spacing to the right (margin-inline-end) and bottom.

<img src="image.jpg" alt="Right-aligned image" class="img-right">

.img-right

Purpose: Displays a half-width image floated to the right (or end of the inline direction).

Behavior: Floats the image to the right side of the container.
Sets the width to the same half-column value.
Adds spacing to the left (margin-inline-start) and bottom.

<img src="image.jpg" alt="Main image" class="img-main">

.img-main

Purpose: Displays a centered main image with a border and responsive sizing.

Behavior:
Centers the image horizontally.
Adds bottom spacing.
Limits width to container size.
Adds a thin border with a gray background color.
Maintains aspect ratio.

Boxes

The CSS provides several boxes: Callout boxes, gray boxes, colored boxes with 2 possible accent colors. Furtherore the CSS allows to put the boxes to the right side, to the left side and at normal size. Some boxes can also be expanded to the full width, so to the borders of the outer area. But let's start with image boxes.

Image boxes

Image boxes are available in two main configurations that are incompatible to each other. One configuration can have image credits and title, the other can have any content, but it looks differently. 

<!-- Image box with credits -->
<div class="image-box">
    <div class="image-content">
        <img src="image.jpg" />
        <div class="image-credits">Image Credits: John Doe</div>
    </div>
    <div class="image-caption">Caption of the image</div>
</div>

<!-- Image box with generic content -->
<div class="image-box">
    <div class="image-content">
        <img src="image.jpg" />
        <div class="image-overlay">
            <div class="image-overlay-content">Caption of the image</div>
            <div class="image-overlay-content">Image Credits: John Doe</div>
        </div>
    </div>
</div>

Callout boxes

If you want to draw attention to a specific paragraph, you can put it into a callout box

<div class="callout-box">There some important text</div>

Gray and accent-colored boxes

There ar also other variants of boxes with a background color, either in gray or in accent colors. You can specify the used gray and accent colors in the CSS. 

<div class="callout-box">Some text with accent</div>

<div class="callout-box">Some text with alternative accent</div>

Additionally there are different variants of the boxes, e.g. normal that have no appendix in name, there are boxes that are aligned to the left with -left in the name, as well as with -right in the name that are alligned to the right.

A variant with -full in the name will take the full width up to the outer area of the page. Not all boxes support the full variant, e.g. callout boxes don't.

In this page you have seen many gray boxes, that are centered, aligned to left or right. This is how it looks like.

Notches

Notches are boxes with a visible border on two sides. For western raders that have left-to-right writing direction the notch will be in left-upper corner. 

<div class="notch">Some text that is set apart</div>

There is also a notch with a thick border you can see it there:

<div class="notch-strong">Some text that is set apart</div>

Grid cards

You can put some content on smaller cards and let the browser place the cards on a grid, responsive to the size of visitor's viewport.
